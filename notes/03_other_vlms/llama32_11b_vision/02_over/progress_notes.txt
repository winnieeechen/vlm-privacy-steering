Llama-3.2-11B-Vision over-disclosure steering notes

Goal:
Reduce over-disclosure for Llama-3.2-11B-Vision using the same original-logic over-disclosure steering pipeline used for Qwen2.5-VL-3B.

Model:
meta-llama/Llama-3.2-11B-Vision-Instruct

Dataset:
Same original-logic full1200 split.

Train: 717
Val: 238
Test: 243

Over-disclosure vector logic:

Condition vector:
positive = true A/B
negative = true C

Purpose:
Detect whether the image-question pair is privacy-sensitive, i.e. whether exact location disclosure should not be allowed.

Behavior vector:
positive = A_to_A, B_to_B
negative = A_to_B, A_to_C, B_to_C

Purpose:
Move generation away from over-disclosure behavior and toward correct privacy-preserving behavior.

------------------------------------------------------------
Layer sweep observation
------------------------------------------------------------

Layer 24 is the best among the tested layer-32-neighborhood settings, but the improvement is still very small:
Correct +2, over-disclosure -1, under-disclosure -1.

Layers 28, 32, and 36 are nearly identical under conditional routing:
Correct +1, over-disclosure -1, under-disclosure unchanged.

The layer sweep does not reveal a strong steering layer for this behavior vector.

Likely reason:
The over behavior vector is poorly matched to Llama's actual error pattern.
In the train behavior-vector table, the positive side contains only B_to_B:
positive: B_to_B = 114
negative: A_to_B = 258, A_to_C = 32, B_to_C = 29

There are no A_to_A examples in the positive behavior set.
Therefore the vector is not really learning a refusal / abstention direction for true-A cases.
It mostly learns a direction that pulls outputs away from C and toward B, which explains why steering often converts C_to_C into C_to_B and increases under-disclosure.

Code check:
The steering script uses the expected Llama model path:
model.model.language_model.layers[layer]

It applies the vector to the last token hidden state in the hooked decoder layer, matching the Qwen steering pattern:
hidden[:, -1, :] = hidden[:, -1, :] + alpha * vector

The input formatting uses MllamaForConditionalGeneration plus AutoProcessor and the same prompt as base inference.
No obvious implementation bug was found in the steering hook or path.
