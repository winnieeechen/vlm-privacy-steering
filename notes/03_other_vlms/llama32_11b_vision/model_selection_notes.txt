New VLM experiment: Llama-3.2-11B-Vision

Goal:
Run the same VLM-GeoPrivacy activation steering pipeline on a second VLM.

Motivation:
The previous Qwen2.5-VL-3B baseline produced much more under-disclosure than over-disclosure.
We want to test a model with the opposite tendency: more over-disclosure than under-disclosure.

Reference model choice:
The VLM-GEOPRIVACY paper reports that Llama-4-Maverick shows more over-disclosure than under-disclosure in vanilla free-form prompting.
However, Llama-4-Maverick is very large and may be difficult to run on our current GPU resources.

Practical model choice:
meta-llama/Llama-3.2-11B-Vision-Instruct

Reason:
It is an open-weight VLM and more practical than Llama-4-Maverick.
We will first run base inference to check whether it shows the desired over-disclosure-heavy behavior on our A/B/C prompt.

Folder structure:
scripts/03_other_vlms/llama32_11b_vision
outputs/03_other_vlms/llama32_11b_vision
notes/03_other_vlms/llama32_11b_vision

Plan:
1. Create base inference script for Llama-3.2-11B-Vision-Instruct.
2. Run val/test base inference.
3. Check correct / over / under.
4. Continue with vector extraction and steering only if the baseline behavior is useful.
